**问题：** 首页「今日餐食」卡片仍不显示食物描述和三大营养素（蛋白质/碳水/脂肪）。

**根因：**
- 数据库中部分老记录的 `total_protein`/`total_carbs`/`total_fat` 为 null
- 后端 `backend/main.py` 用 `float(x.get("total_protein") or 0)` 聚合，导致营养素变成 0
- 前端 `home_dashboard_local_cache_v3` 缓存了这些 0 值，且渲染条件写的是 `Number(meal.protein) > 0`，所以即使缓存通过校验，值为 0 时也不显示

**修复：**
1. 后端增加兜底计算：当顶层宏量营养素为 0 时，从 `record.items[].nutrients` 按 `ratio` 重新聚合
2. 前端放宽渲染条件为 `typeof meal.protein === 'number'`，有定义就显示
3. 前端缓存读取增加临时迁移：检测到「有 description 但三大营养素全为 0」的快照时，强制清空脏缓存

### 修复：运动后返回首页运动卡片仍显示 0

**问题根因分析：**
1. `src/pages/exercise-record/index.tsx` 在成功记录运动后会触发 `HOME_DASHBOARD_REFRESH_EVENT`
2. 首页 `src/pages/index/index.tsx` 的 `useDidShow` 监听该事件并设置 `homeDataStaleRef.current = true`
3. 但 `useDidShow` 中的刷新逻辑有一个限制条件：`if (currentSelected === today || !currentSelected)`，**只有当首页当前选中日期是"今天"时才刷新**
4. 如果用户之前在首页切到了其他日期（如昨天），然后去记录运动，再回到首页时，由于 `currentSelected !== today`，`useDidShow` 会直接跳过刷新，导致运动卡片仍显示旧缓存值（0）
5. 此外，`getExerciseLogs` API 缺少 `_t` 时间戳参数，存在微信小程序 GET 请求缓存风险

**修复内容：**
1. **放宽首页刷新条件**：`useDidShow` 中当 `homeDataStaleRef.current === true`（数据变脏）时，无论当前选中日期是不是今天，都会刷新 dashboard
2. **清除运动 API 缓存**：`src/utils/api.ts` 的 `getExerciseLogs` 追加 `_t=${Date.now()}` 参数，避免微信缓存返回旧数据
