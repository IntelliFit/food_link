- 真机调试新增排查结论（2026-04-08 晚）：
  - Console 中 `request:fail -102:net::ERR_CONNECTION_REFUSED` 的直接原因是当前构建注入的 `API_BASE_URL=http://127.0.0.1:3010`；该地址在真机上指向手机本机，不会连到开发电脑后端。
  - 另有独立前端运行时异常 `ReferenceError: performance is not defined`，已在源码定位到 `src/pages/index/hooks/useAnimatedProgress.ts` 与 `src/pages/index/hooks/useAnimatedNumber.ts` 对 `performance.now()` 的直接调用（小程序 JSCore 环境下不稳定）。
  - 结论：当前报错是“网络依赖 + 运行时依赖”叠加，不是单一后端接口问题。

- 同轮代码修复（2026-04-08 晚）：

- 运动记录 0 kcal bug（2026-04-08 凌晨）：

    - 若仍无法可靠解析，不再写 `0`，而是让任务失败，前端显示失败态，避免假成功。
    - `backend/worker.py` 已增加运动任务开始/完成/失败日志，便于直接在终端看链路。
