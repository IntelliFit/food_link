- 之后已直接查库并复现截图对应任务 `68e994ad-73ba-42be-9cc3-8c7e36e3a061`：
  - 任务类型：`food_text_debug`
  - 文本：`我今天负重60千克做了5组5次的引体向上  然后还做了10几个单手引体`
  - 画像快照：`70.2kg / 180cm / 男 / 24岁 / moderate / BMR 1710 / TDEE 2650.5`
- 精确根因：
  - 同一条描述 + 同一快照在本地直接复现时，Instructor 连续 3 次都报 `The output is incomplete due to a max_tokens length limit`
  - `google/gemini-3-flash-preview` 在当前 `Instructor MD_JSON + max_tokens=900` 组合下，内部 reasoning token 膨胀，导致结构化 JSON 还没输出完整就被截断
  - 最终被包装成 `InstructorRetryException`，再被业务层统一转成“运动热量估算失败，请稍后重试”
- 对照样本：
  - 不带画像快照时，同句描述本地能成功返回 `185 kcal`
  - 说明问题不是“这句话不会算”，而是“带完整画像上下文时，当前模型/模式组合在这类高强度力量描述下更容易被 token 截断”
