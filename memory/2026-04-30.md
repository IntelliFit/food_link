  - 关键链路可上报 trace
  - 日志可通过 OTel 上报
  - 把 trace id 回传到 HTTP 响应头，便于前端反馈问题时快速定位
- 已在 `backend/main.py` 落地：
  - OTel trace：`TracerProvider + OTLPSpanExporter + FastAPIInstrumentor + HTTPXClientInstrumentor`
  - OTel logs：`LoggerProvider + OTLPLogExporter + LoggingHandler`
  - 响应头注入：`x-trace-id`、`traceparent`，并新增 `x-instance-id`
- 实例标识头规则：
  - 默认响应头名：`x-instance-id`（可用 `INSTANCE_HEADER_NAME` 覆盖）
  - 值优先取 `POD_NAME`，回退 `HOSTNAME`，最后兜底 `unknown-instance`
  - 可通过 `INSTANCE_HEADER_ENABLED` 开关控制
- 已在 `backend/.env` 增加默认配置：
  - `OTEL_ENABLED=1`
  - `OTEL_LOGS_ENABLED=1`
  - `OTEL_SERVICE_NAME=food-link-backend`
  - `OTEL_EXPORTER_OTLP_ENDPOINT=http://otel-collector.observability.svc.cluster.local:4318`
  - `INSTANCE_HEADER_ENABLED=1`
