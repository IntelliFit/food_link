## Go backend cmd/server vs cmd/worker

- User asked what the difference is between `backend/cmd/server` and `backend/cmd/worker`, and whether `worker` can be removed.
- Current code facts:
  - `cmd/server` is the real HTTP API entrypoint: loads config, builds `internal/app`, starts `http.Server`, handles graceful shutdown, and closes app resources.
  - At the time of review, `cmd/worker` was only a placeholder bootstrap: loaded config, logged env/database target, then slept forever.
- Initial recommendation was to keep it if a separate async runtime was still planned, but the user then explicitly chose to delete it.
