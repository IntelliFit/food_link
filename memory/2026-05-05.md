## Membership credit detail adjustments

- User reported manual upgrades to the second user's membership credits kept reverting.
- Cause: `_reconcile_membership_from_latest_paid_order()` repairs `user_pro_memberships` from the latest real paid order and previously treated any `daily_credits` mismatch as drift, so manual boosts were overwritten to the paid plan default.
- New rule:
  - Payment truth still controls plan code, status, paid time, period start, and expiry.
  - `daily_credits` may be manually boosted above the plan expected value and should be preserved.
  - Values below the paid-plan expected value are still repaired upward.
  - `backend/scripts/reconcile_membership_truth.py` uses the same rule.
