## Food photo analysis algorithm review

- User asked for an analysis of the photo-based food analysis algorithm.
- Code was reviewed without making algorithm changes.
- Current implementation is a hybrid pipeline:
  - Standard image mode defaults to `analysis_engine=db_first`: vision model recognizes food names and weights, backend nutrition library supplies nutrients, and DeepSeek fills unknown per-100g nutrition then writes it back for future reuse.
  - Strict/precision mode uses a multi-step pipeline: planner splits visible foods, item-estimate tasks estimate weights, optional weight refinement rechecks high-uncertainty items, and aggregate combines item nutrients through the same library/fallback path.
- Noted review point: standard `/api/analyze/submit` normal-task credit consumption path should be rechecked because the visible code path passes precision cost after creating a standard task.
