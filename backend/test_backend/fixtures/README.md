# 食物识别标准测试集

测试后台的批量测试包使用 ZIP 格式，推荐结构：

```text
standard-set.zip
├── bento01.jpg
├── bento02.jpg
└── labels.txt
```

`labels.txt` 支持两种标准标签模式。

如果只有整餐总重量，使用总重量模式；后台只评估“模型估算总重量 vs 标准总重量”：

```text
bento01.jpg 500g
bento02.jpg | 总重量=620g
```

如果知道每种食物的标准重量，使用逐项模式；后台会同时评估总重量偏差和每个食物的匹配/克重偏差：

```text
bento03.jpg | 米饭=120g; 鸡胸肉=80g; 西兰花=60g
bento04.jpg | 鸡蛋=55g; 全麦面包=70g; 牛奶=240g
```

建议标准测试集分层：

- `single-clear`: 单一食物、边界清楚，用于测基础估重。
- `multi-item`: 一餐多食物，用于测“识别对不对 + 每项重量对不对”。
- `hand-reference`: 带手、筷子、勺子、碗盘等参照物，用于测参照物利用。
- `multi-view`: 同一份食物多角度图，用于测多视角模式。
- `hard-cases`: 遮挡、酱汁、混合菜、深碗等难例，用于回归测试。

每次改 prompt 或模型后，优先用同一批 ZIP 同时勾选 Qwen / Gemini 对比，记录每个模型的总偏差和逐项偏差。
