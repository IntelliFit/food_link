# 首页产品需求文档 (PRD)

> 文档版本: v1.0  
> 更新日期: 2026-03-28  
> 产品名称: 食探 - AI 智能饮食记录

---

## 1. 页面概述

首页是「食探」小程序的核心入口页面，主要承载用户每日饮食数据的概览展示、快捷记录入口以及目标管理功能。

---

## 2. 数据展示功能

### 2.1 问候区

| 数据项 | 说明 | 数据来源 |
|--------|------|----------|
| 时间问候语 | 根据当前时间段显示「早上好」/「下午好」/「晚上好」 | 系统时间（<12点早上，<18点下午，其他晚上） |
| 鼓励文案 | 固定显示「今天也要健康饮食哦」 | 静态文案 |

### 2.2 今日摄入卡片

| 数据项 | 说明 | 数据来源 |
|--------|------|----------|
| 当前摄入热量 | 今日已摄入的总卡路里 | API: `getHomeDashboard` |
| 目标热量 | 每日热量目标值，默认 2000 kcal | API + 本地存储 |
| 完成进度百分比 | (当前摄入/目标) × 100% | 前端计算 |
| 剩余可摄入热量 | 目标 - 当前摄入（最小为0） | 前端计算 |
| 进度条 | 可视化展示完成比例 | 前端渲染 |

### 2.3 三大营养素展示

| 数据项 | 说明 | 目标默认值 | 数据来源 |
|--------|------|------------|----------|
| 蛋白质 | 当前摄入/目标值，环形进度图 | 120g | API: `getHomeDashboard` |
| 碳水化合物 | 当前摄入/目标值，环形进度图 | 250g | API: `getHomeDashboard` |
| 脂肪 | 当前摄入/目标值，环形进度图 | 65g | API: `getHomeDashboard` |

**展示样式**：
- 蛋白质：蓝色 (#3b82f6)
- 碳水：绿色 (#00bc7d)
- 脂肪：橙色 (#f59e0b)

### 2.4 本周饮食热力图

| 数据项 | 说明 | 数据来源 |
|--------|------|----------|
| 7天日期格子 | 展示最近7天的饮食记录状态 | API: `getStatsSummary('week')` |
| 记录状态 | 灰色=未记录，红色=热量盈余，蓝色=热量不足 | 根据与 TDEE 差值计算 |
| 颜色深浅 | 根据偏离程度分 3 个等级 | deltaRatio > 30% 最深 |
| 记录天数统计 | 显示「本周已记录 X/7 天」 | 前端计算 |

**热力图颜色规则**：

| 状态 | 条件 | Level 1 | Level 2 | Level 3 |
|------|------|---------|---------|---------|
| none | 无记录 | 灰色半透明 | - | - |
| surplus | 摄入 > TDEE | 浅红 | 中红 | 深红 |
| deficit | 摄入 < TDEE | 浅蓝 | 中蓝 | 深蓝 |

### 2.5 今日餐食列表

| 数据项 | 说明 | 数据来源 |
|--------|------|----------|
| 餐次类型 | 早餐、上午加餐、午餐、下午加餐、晚餐、晚上加餐、零食 | API: `getHomeDashboard` |
| 餐次名称 | 该餐次的描述名称 | API |
| 记录时间 | 餐食记录的时间 | API |
| 摄入热量 | 该餐次的卡路里 | API |
| 目标占比 | 该餐占每日目标的百分比 | 前端计算 |
| 餐次标签 | 如「均衡」、「蛋白质丰富」等标签 | API |

**餐次图标配色**：

| 餐次 | 图标 | 颜色 |
|------|------|------|
| 早餐 | 早餐图标 | 橙色 (#ff6900) |
| 上午加餐 | 零食图标 | 紫色 (#7b61ff) |
| 午餐 | 午餐图标 | 绿色 (#00c950) |
| 下午加餐 | 零食图标 | 紫色 (#ad46ff) |
| 晚餐 | 晚餐图标 | 蓝色 (#2b7fff) |
| 晚上加餐 | 零食图标 | 深紫 (#5b21b6) |
| 零食 | 零食图标 | 紫色 (#ad46ff) |

## 3. 交互功能

### 3.1 目标管理

| 功能 | 说明 | 触发方式 | 限制条件 |
|------|------|----------|----------|
| 打开目标编辑器 | 弹出底部模态框编辑目标 | 点击「编辑目标」按钮 | 需登录状态 |
| 修改热量目标 | 输入新的每日热量目标 | 数字输入框 | 500-6000 kcal |
| 修改蛋白质目标 | 输入新的蛋白质目标 | 数字输入框 | 0-500 g |
| 修改碳水目标 | 输入新的碳水目标 | 数字输入框 | 0-1000 g |
| 修改脂肪目标 | 输入新的脂肪目标 | 数字输入框 | 0-300 g |
| 自动校准 | 修改热量时按比例调整营养素，反之自动计算热量 | 实时计算 | - |
| 保存目标 | 校验并保存目标值 | 点击「保存」按钮 | 需通过范围校验 |

**目标值校验规则**：

```
热量目标: 500 ≤ value ≤ 6000 (kcal)
蛋白质: 0 ≤ value ≤ 500 (g)
碳水: 0 ≤ value ≤ 1000 (g)
脂肪: 0 ≤ value ≤ 300 (g)

热量换算公式: 热量 = 蛋白质×4 + 碳水×4 + 脂肪×9
保存时会以热量目标为准，自动校准三大营养素
```

### 3.2 快捷记录

| 功能 | 说明 | 触发方式 | 限制条件 |
|------|------|----------|----------|
| 拍照识别 | 通过拍照识别食物并记录饮食 | 点击「拍照识别」按钮 | **必须登录** |
| 文字记录 | 通过文字输入描述饮食 | 点击「文字记录」按钮 | 无限制 |

**跳转逻辑**：
- 拍照识别：设置 `recordPageTab = 'photo'`，跳转 `/pages/record/index`
- 文字记录：设置 `recordPageTab = 'text'`，跳转 `/pages/record/index`
- 未登录时点击拍照识别：跳转 `/pages/login/index`

### 3.3 记录查看

| 功能 | 说明 | 触发方式 |
|------|------|----------|
| 查看饮食记录 | 跳转到统计页面查看详细记录 | 点击「饮食记录」链接 |
| 查看今日餐食 | 跳转到记录页面查看全部餐食 | 点击「今日餐食记录」链接 |
| 查看历史日期 | 点击热力图某天查看该日详情 | 点击有记录的热力图格子 |

**跳转参数**：
- 查看今日餐食：设置 `recordPageTab = 'history'`
- 查看历史日期：设置 `recordPageTab = 'history'` + `recordHistoryDate = 日期`

### 3.4 分享功能

| 功能 | 说明 | 触发方式 |
|------|------|----------|
| 分享给好友 | 分享给微信好友/群 | 右上角菜单 → 转发 |
| 分享到朋友圈 | 分享到微信朋友圈 | 右上角菜单 → 分享到朋友圈 |

**分享内容**：
- 标题：「食探 - AI 智能饮食记录」
- 路径：`/pages/index/index`

## 4. 业务逻辑

### 4.1 登录状态处理

| 场景 | 处理逻辑 |
|------|----------|
| 已登录 | 正常加载所有数据，显示真实数据 |
| 未登录 | 显示默认数据（热量目标 2000 kcal），可浏览但拍照识别需登录 |
| 点击编辑目标（未登录） | Toast 提示「登录后可编辑目标」 |
| 点击拍照识别（未登录） | 跳转登录页 |
| 点击饮食记录（未登录） | 跳转登录页 |

### 4.2 数据加载策略

| 场景 | 处理逻辑 |
|------|----------|
| 页面首次加载 | 显示 loading 状态，请求 `getHomeDashboard` 和 `getStatsSummary` |
| 页面再次显示 | 使用 `useDidShow` 钩子自动刷新数据 |
| 网络异常 | 显示默认数据，保持页面可用 |
| 目标编辑后 | 重新加载仪表盘数据，同步最新状态 |

### 4.3 数据存储策略

| 数据类型 | 存储方式 | 说明 |
|----------|----------|------|
| 目标值 | 本地 Storage + 后端同步 | 后端升级前暂存本地，升级后自动同步 |
| 页面Tab状态 | 本地 Storage | 用于向记录页传递参数 |
| 历史日期 | 本地 Storage | 用于向记录页传递选中日期 |

## 6. API 接口

### 6.1 获取首页数据

```typescript
// GET /api/home/dashboard
interface HomeDashboardResponse {
  intakeData: {
    current: number;      // 当前摄入热量
    target: number;       // 目标热量
    progress: number;     // 完成进度 (0-100)
    macros: {
      protein: { current: number; target: number; };
      carbs: { current: number; target: number; };
      fat: { current: number; target: number; };
    };
  };
  meals: HomeMealItem[];
}

interface HomeMealItem {
  type: 'breakfast' | 'morning_snack' | 'lunch' | 'afternoon_snack' | 'dinner' | 'evening_snack' | 'snack';
  name: string;
  time: string;
  calorie: number;
  target: number;
  progress: number;
  tags?: string[];
}
```

### 6.2 获取统计数据

```typescript
// GET /api/stats/summary?range=week
interface StatsSummaryResponse {
  daily_calories: {
    date: string;      // YYYY-MM-DD
    calories: number;
  }[];
  tdee: number;        // 每日总能量消耗
}
```

### 6.3 更新目标

```typescript
// POST /api/home/targets
interface UpdateDashboardTargetsPayload {
  calorie_target: number;
  protein_target: number;
  carbs_target: number;
  fat_target: number;
}

interface UpdateDashboardTargetsResponse {
  saveScope: 'local' | 'remote';  // local=暂存本地, remote=已同步云端
}
```
